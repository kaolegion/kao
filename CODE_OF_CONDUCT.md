# KAO — Code of Conduct

## Intent

Kao is built as an open, rigorous and constructive project.

This code of conduct exists to protect:

- clarity
- respect
- collaboration quality
- healthy technical exchange
- sustainable project growth

---

## Expected behavior

Contributors are expected to:

- communicate respectfully
- critique ideas without attacking people
- stay constructive during disagreement
- keep discussions grounded in observable facts
- value documentation, tests and clarity
- help preserve a readable and operable project culture

---

## Unacceptable behavior

The following behaviors are not acceptable:

- harassment or intimidation
- personal attacks
- contemptuous or degrading language
- repeated disruptive behavior
- intentionally obscuring technical reality
- sabotaging documentation, tests or collaboration flow

---

## Kao project culture

Kao favors a culture of:

- explicit reasoning
- inspectable change
- small controlled steps
- calm technical correction
- convergence between code, docs and tests

This applies both to implementation work
and to discussions around the project.

---

## Conflict handling

When disagreement happens, contributors should try to:

1. restate the technical issue clearly
2. inspect the relevant files or behavior
3. compare concrete evidence
4. prefer the most coherent and testable path
5. document the outcome when needed

---

## Enforcement

Project maintainers may moderate or restrict participation
when behavior damages collaboration quality,
project readability, or contributor safety.

The goal is not punishment for its own sake.

The goal is to preserve a healthy operator culture.

---

## Final principle

Kao is not only built through code.

It is also built through the quality
of the way people think, explain, review and collaborate together.
