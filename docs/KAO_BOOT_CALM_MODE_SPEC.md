# KAO — BOOT CALM MODE SPEC

## Purpose

This document defines the minimal calm boot perception of Kao.

It introduces a reduced manifestation mode intended for:

- daily operator usage
- stable long-term perception
- low cognitive noise environments
- mature Kao instances

The calm mode does not replace the full scene.
It complements it.

---

## 1 — Calm mode philosophy

Kao should not always speak extensively.

A mature system manifests with restraint.

Calm mode aims to transmit:

- presence
- continuity
- stability

with the smallest possible signal.

---

## 2 — Canonical calm scene

Recommended minimal structure:

Kao : present  
Runtime : stable  
Memory : continuity | genesis  
Gateway : ready | degraded | offline  

Operator prompt follows immediately.

---

## 3 — Temporal behavior

Calm mode boot must feel:

- immediate
- silent
- confident

No banner.
No large layout.
No decorative framing.

---

## 4 — Activation conditions

Calm mode may be enabled when:

- runtime consistency is strong
- identity resolution is stable
- previous sessions exist
- operator familiarity is assumed

Future heuristics may include:

- system age
- heat maturity
- error history
- operator preference

---

## 5 — Degraded override

If instability is detected:

- Kao must fall back automatically
- full diagnostic scene becomes visible

Calm mode must never hide risk signals.

---

## 6 — Long-term product role

Calm mode represents:

- system maturity
- trust relationship
- cognitive minimalism

It supports the perception of Kao as:

a quiet but reliable presence.

---

## Canonical principle

At first, Kao explains.

Later, Kao simply appears.
