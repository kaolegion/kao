# KAO — Operator Guide (Digest)

This document provides a clear operational entrypoint
for using Kao as a cognitive terminal system.

It synthesizes onboarding and runtime manual principles.

---

## First contact

Kao starts in a terminal scene.

Before acting, the operator must:

- read the system scene
- understand active authority
- evaluate runtime stability

Safe operation begins with perception.

---

## Safe first actions

Recommended initial workflow:

- inspect system status
- observe runtime signals
- review memory continuity
- avoid immediate transitions

Read commands before action commands.

---

## Runtime hygiene

Kao distinguishes:

- versioned source (code, docs, libraries)
- ephemeral runtime state (logs, snapshots, diagnostics)

Operator rule:

runtime mutation is normal  
runtime pollution of source is not.

A clean repository after validation is a stability signal.

---

## Hybrid routing perception

The operator must understand execution context:

- local-only cognitive mode
- hybrid network mode
- cloud-extended cognition

Routing decisions depend on:

- system state
- resource availability
- task nature

Kao remains state-aware.

---

## Agent supervision

Agents are not invisible assistants.

They must be:

- observable
- interpretable
- correctable
- stoppable

Operator lucidity is the primary safety mechanism.

---

## Session cognition

Kao sessions evolve:

OPEN → LIVE → CLOSE → MEMORY → SUMMARY

Operator should:

- close sessions consciously
- read summaries
- maintain cognitive continuity

---

## Operational mindset

Kao is not used like a tool.

It is inhabited as an operational environment.

The operator learns to:

- read signals
- pace actions
- maintain system clarity
- collaborate with cognition layers
