# KAO — BOOT TERMINAL SCENE

## Purpose

This document defines the canonical terminal perception of Kao at activation.

It specifies the rhythm, structure, and emotional neutrality of the boot scene.

The goal is to make Kao feel:

- present
- readable
- calm
- alive but minimal

---

## 1 — Boot philosophy

Boot must not feel like a noisy system start.

Boot must feel like a controlled manifestation.

Silence is part of cognition.

Output must remain intentional.

---

## 2 — Temporal rhythm

Recommended perception sequence:

1. short silence (context inspection)
2. identity emergence
3. runtime signal
4. cognitive readiness

Boot must not exceed a few seconds.

---

## 3 — Canonical minimal scene

Example structure:

Machine   : <host>
System    : <environment>
Operator  : <user>
Runtime   : stable
Memory    : detected | genesis
Gateway   : ready | degraded | offline

Kao       : present

---

## 4 — Emotional neutrality

The scene must:

- avoid marketing tone
- avoid dramatic vocabulary
- avoid artificial friendliness

It must transmit:

- technical calm
- reliability
- continuity

---

## 5 — Cognitive signal

After the scene, Kao may expose:

- cockpit summary
- last session age
- heat level
- operator prompt readiness

This confirms cognition is active.

---

## 6 — Long-term evolution

Future boot scenes may include:

- adaptive verbosity
- maturity signals
- distributed presence hints
- agent readiness signals

But the canonical minimal scene must remain valid forever.

---

## Canonical principle

Kao does not greet.

Kao appears.
