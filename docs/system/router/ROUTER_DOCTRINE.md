# KAO — Router Doctrine

The Router is the decision cortex of Kao.

It is responsible for orienting execution flows
between internal system capabilities, agents and external intelligence.

---

## Nature

The Router is not an assistant.
The Router is not an agent.

The Router is a systemic arbitration mechanism.

It evaluates:

- system state
- task nature
- operator intent
- resource availability
- cognitive continuity

---

## Core responsibilities

- choose execution mode
- select agents
- select providers
- preserve system stability
- maintain sovereignty boundaries

---

## Execution modes

### Local cognitive mode

- no network dependency
- internal tools and local models
- maximum sovereignty

### Hybrid mode

- local runtime with selective external assistance
- controlled agent orchestration

### Extended cognition mode

- distributed agents
- cloud intelligence
- increased capability with supervision

Router must always prefer the most stable viable mode.

---

## Arbitration principles

Router decisions must remain:

- observable
- explainable
- reversible when possible

Opaque routing is considered a system failure.

---

## Cognitive continuity

Router must preserve:

- session coherence
- memory traceability
- operator readability

Routing that breaks cognitive continuity
must be avoided unless required for system safety.

---

## Long-term evolution

Router may evolve toward:

- adaptive policy learning
- multi-node cognitive arbitration
- context heat awareness
- strategic execution pacing

But core doctrine remains:

Kao decides. Agents execute.
