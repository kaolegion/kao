# KAO METRICS BASELINE

## Purpose

Kao must remain sensitive to its environment.

This baseline introduces a first canonical metrics surface so Kao can observe
hardware and runtime signals before deeper cognition and visual materialization.

## Command surface

The baseline metrics surface is exposed through:

- `kao metrics`

## Metrics captured

The current baseline captures:

- host
- kernel
- uptime
- load average
- total memory
- available memory
- root disk usage
- default route presence
- active session id
- markdown documentation count

## Product role

These metrics are the first perception layer for:

- hardware awareness
- software/runtime awareness
- documentation growth tracking
- future cockpit and alerting surfaces

## Future evolution

This baseline should later expand toward:

- provider health
- router decision counters
- runtime consistency counters
- session frequency
- timeline event counts
- file/system change signals
