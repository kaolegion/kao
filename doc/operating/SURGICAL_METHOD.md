# KAO — Surgical Operating Method

## Purpose

Kao evolves through disciplined structural intervention.

The surgical method defines how an operator safely modifies
a living cognitive system.

It ensures clarity, reproducibility and sovereign control.

---

## Phase 1 — Sentinel  
Structural verification and system perception.

- inspect targeted files
- scan surrounding architecture
- confirm execution surfaces
- detect anomalies

Nothing is modified during this phase.

---

## Phase 2 — Rogue  
Reality exposure and signal acquisition.

- read logs
- run baseline tests
- observe runtime behavior
- identify friction zones

Assumptions must be replaced by observable facts.

---

## Phase 3 — Syfer  
Minimal deterministic source intervention.

- inject structural changes
- update modules or scripts
- align execution permissions when required
- perform atomic modifications

Correctness is prioritized over speed.

---

## Phase 4 — Ava  
Focused validation of the mutation.

- run targeted verification commands
- inspect outputs
- confirm behavioral change
- reopen modified surfaces

If validation fails, the loop returns to Sentinel.

---

## Phase 5 — Convergence  
System reintegration and stabilization.

- update documentation
- perform end-to-end tests
- clean temporary artifacts
- ensure working tree clarity
- commit with explicit intent
- push and perform post-check

Only after convergence is the intervention considered complete.

---

## Final Principle

Kao does not evolve through improvisation.

Kao evolves through disciplined surgical operations.
