# KAO — Changelog

## Intent

This file records notable project evolution in a compact,
readable and operator-friendly way.

It is not meant to replace detailed sprint notes.

It is meant to provide a clear historical surface
for major structural, behavioral and documentation changes.

---

## Changelog doctrine

A changelog entry should favor:

- clarity
- compact wording
- observable change
- concrete scope
- readable chronology

Entries should summarize what changed,
not every command used during implementation.

---

## 0.1.0-doc-foundation

### DOC 1.x — Documentation foundation series

Established the first canonical project documentation layer:

- founding vision pack under `docs/`
- root project entrypoints in `README.md`
- contribution guidance in `CONTRIBUTING.md`
- roadmap foundation in `ROADMAP.md`
- conduct policy in `CODE_OF_CONDUCT.md`
- security policy in `SECURITY.md`
- support guidance in `SUPPORT.md`
- project map in `PROJECT.md`
- release doctrine in `RELEASE.md`
- canonical version file in `VERSION`

This series created the first coherent documentation spine
for Kao as a terminal-first cognitive operator project.
