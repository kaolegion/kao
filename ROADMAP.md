# KAO — Roadmap Foundation

## Current direction

Kao is evolving toward a cognitive operator environment
structured around runtime identity, memory, terminal UX,
agent orchestration and development intelligence.

---

## Near-term priorities

### 1. Runtime identity consolidation

Stabilize and clarify:

- machine
- system
- owner
- user

Goals:

- deterministic activation
- readable runtime states
- canonical command paths
- explicit source policies

### 2. Terminal UX doctrine

Strengthen the terminal as primary interface.

Goals:

- readable login scenes
- compact operator output
- better state visualization
- clearer runtime narratives

### 3. Documentation convergence

Build a coherent project foundation.

Goals:

- vision-first entry
- contribution guide
- architecture clarity
- roadmap continuity

---

## Mid-term priorities

### 4. Kao Brain foundation

Prepare the memory layer.

Directions:

- context persistence
- graph memory
- timeline reasoning
- skill and trace structures

### 5. Agent orchestration foundation

Prepare supervised agent execution.

Directions:

- tool-aware agents
- deterministic task routing
- observable agent states
- human supervision paths

### 6. Dev intelligence layer

Expand repository cognition.

Directions:

- repository mapping
- architecture reading
- diff intelligence
- sprint assistance

---

## Long-term direction

### 7. Cognitive operating system maturity

Kao aims to become:

- a terminal cognitive environment
- a memory-aware operator system
- a supervised agent runtime
- a development cognition platform
- an open modular architecture for human-agent collaboration

---

## Roadmap doctrine

Kao should evolve by keeping:

- explicit layers
- controlled runtime states
- visible policies
- documentation alignment
- operator-readable behavior
