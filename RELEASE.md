# KAO — Release Doctrine

## Intent

This document defines the initial release and versioning doctrine for Kao.

It complements:

- `VERSION`
- `CHANGELOG.md`
- `PROJECT.md`

---

## Version file

The canonical current project version is stored in:

- `VERSION`

This file should stay compact and machine-readable.

---

## Release doctrine

A release should be understandable through:

- a version value in `VERSION`
- a readable summary in `CHANGELOG.md`
- aligned documentation when behavior or structure changed

---

## Initial release foundation

Current foundation version:

- `0.1.0-doc-foundation`

Meaning:

- first coherent project documentation spine exists
- project entrypoints are defined
- vision, governance and support documents are present
- changelog and project map are in place

---

## Release expectations

Before declaring a new release, verify:

- important docs are aligned
- canonical paths are still correct
- operator doctrine remains coherent
- changelog is updated for notable changes
- project map still reflects reality

---

## Maintenance note

As Kao evolves, this release doctrine can later expand toward:

- tagged releases
- milestone mapping
- runtime release notes
- compatibility notes
