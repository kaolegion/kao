# KAO — Contributing

## Philosophy

A contribution must improve clarity.

Kao values:

- small changes
- explicit behavior
- documentation convergence
- deterministic runtime evolution

## Expected workflow

Before modifying:

- inspect target files
- understand current runtime behavior
- verify documentation alignment

After modifying:

- re-inspect files
- verify behavior
- update documentation if needed

## Golden rule

A change must make Kao easier to understand,
not harder to reason about.
